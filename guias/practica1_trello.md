# 📝 Práctica 1: Configuración de Trello desde 0 (Guía de Laboratorio)

Esta actividad simula las tareas de preparación que ocurren durante el Esprint Cero, donde el probador colabora para crear el tablero de tareas.

## Paso 1: Creación del Espacio de Trabajo

Ingresa a Trello.com y crea una cuenta gratuita.

Crea un nuevo "Espacio de Trabajo" con el nombre de tu proyecto de curso.

Invita a tus compañeros de equipo (simulando la ubicación común y colaboración).

## Paso 2: Creación del Tablero

Haz clic en "Crear tablero nuevo".

Título: "Sprint 1 - Proyecto Agile Testing".

Visibilidad: Espacio de Trabajo (para fomentar la transparencia).

## Paso 3: Estructura de Columnas (Flujo de Trabajo)

Crea las siguientes columnas de izquierda a derecha para visualizar el flujo de valor:

Backlog del Producto: Para historias de usuario priorizadas por el PO.

Por hacer (To Do): Tareas seleccionadas para el esprint actual.

En curso (WIP): Trabajo activo. Nota: Recuerda limitar el trabajo en curso para optimizar el flujo.

Pruebas / Verificación: Columna específica para que el probador valide los criterios de aceptación.

Hecho (Done): Solo tareas que cumplan con la Definición de Hecho (DoD).

## Paso 4: Creación de Etiquetas (Coordinación por Colores)

Utiliza colores para determinar el tipo de tarea y mejorar la representación visual:

Verde: Historias de Usuario.

Azul: Tareas de Desarrollo.

Naranja: Tareas de Prueba (Manuales/Exploratorias).

Rojo: Defectos / Bugs.

## Paso 5: Primera Tarjeta de Prueba (Ejemplo)

En "Por hacer", crea una tarjeta: "TP-01: Diseñar Casos de Prueba para Login".

Dentro de la tarjeta, añade un Checklist con los Criterios de Aceptación.

Asigna un responsable (el probador).

Mueve la tarjeta a "En curso" para simular el inicio de la actividad.

Esta configuración inicial permitirá que, en las siguientes sesiones, el equipo pueda realizar el seguimiento de las pruebas automatizadas y manuales de forma integrada.
