# 📝 Guía de Laboratorio: Configuración de Trello desde Cero
## Módulo: Agile Testing y Gestión de Proyectos en el Sprint Cero

Esta actividad simula las tareas de preparación que ocurren durante el Esprint Cero, donde el probador colabora activamente para estructurar el flujo de valor y visualización del tablero de tareas.

---

## 🎯 Objetivos de Aprendizaje
*   **Visualización de Flujo (Kanban):** Entender cómo mapear las etapas de desarrollo y pruebas para garantizar la transparencia del sprint.
*   **Coordinación Visual:** Utilizar el código de colores para categorizar los diferentes tipos de historias, tareas y defectos.
*   **Transparencia (Agile):** Compartir el estado del trabajo en tiempo real con todo el equipo.

⏱️ **Duración Estimada:** 45 Minutos

---

## 🛠️ Paso 1: Creación del Espacio de Trabajo
1.  Ingresen a [Trello.com](https://trello.com) y creen una cuenta gratuita si aún no disponen de una.
2.  Creen un nuevo **"Espacio de Trabajo"** asignándole el nombre de su proyecto de curso.
3.  Inviten a sus compañeros de equipo, simulando la ubicación común y la colaboración activa en el proyecto.

## 📋 Paso 2: Creación del Tablero
1.  Hagan clic en **"Crear tablero nuevo"**.
2.  Asignen el título: `"Sprint 1 - Proyecto Agile Testing"`.
3.  Configuren la visibilidad en modo **Espacio de Trabajo** para fomentar la transparencia y el acceso a toda la información.

## ⚙️ Paso 3: Estructura de Columnas (Flujo de Trabajo)
Creen las siguientes columnas de izquierda a derecha para visualizar el flujo de valor de la iteración:
*   **Backlog del Producto:** Para historias de usuario priorizadas por el Product Owner (PO).
*   **Por hacer (To Do):** Tareas e historias seleccionadas y comprometidas para el esprint actual.
*   **En curso (WIP):** Trabajo activo. *Nota: Recuerden limitar el trabajo en curso para optimizar el rendimiento del equipo.*
*   **Pruebas / Verificación:** Columna específica para que el probador valide que se cumplen todos los criterios de aceptación.
*   **Hecho (Done):** Reservado únicamente para las tareas que cumplan plenamente con la Definición de Hecho (DoD).

## 🏷️ Paso 4: Creación de Etiquetas (Coordinación por Colores)
Utilicen el sistema de etiquetas de Trello para categorizar visualmente el tipo de trabajo:
*   🟢 **Verde:** Historias de Usuario (Requisitos de negocio).
*   🔵 **Azul:** Tareas de Desarrollo (Codificación, diseño técnico).
*   🟠 **Naranja:** Tareas de Prueba (Casos de prueba, exploratorias, automatización).
*   🔴 **Rojo:** Defectos / Bugs detectados durante la iteración.

## 🧪 Paso 5: Primera Tarjeta de Ejemplo
1.  En la columna **"Por hacer"**, creen una tarjeta con el título: `"TP-01: Diseñar Casos de Prueba para Login"`.
2.  Añadan la etiqueta **naranja** (Pruebas).
3.  Añadan un Checklist detallado con los criterios de aceptación que se deben verificar.
4.  Asignen la tarjeta a un miembro del equipo (el probador).
5.  Muevan la tarjeta a la columna **"En curso"** para reflejar el inicio real del trabajo.

> [!TIP]
> Esta configuración inicial permitirá que, en las siguientes sesiones, el equipo pueda realizar el seguimiento de las pruebas automatizadas y manuales de forma integrada y sin silos de información.

---

## 📝 Preguntas de Reflexión y Análisis Crítico (Para los Estudiantes)
1.  ¿Por qué es importante establecer una columna específica para "Pruebas / Verificación" en lugar de mover el trabajo directamente de "En curso" a "Hecho"?
2.  ¿Cómo apoya la limitación del Trabajo en Curso (WIP) a la productividad y a la detección temprana de cuellos de botella?
